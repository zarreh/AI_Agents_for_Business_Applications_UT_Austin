# AI Agent Instructions for Agentic AI for Business Applications

## Project Overview
This is a **learning-focused repository** for the UT Austin × Great Learning course: **"Agentic AI for Business Applications"**. It teaches building LLM-powered agents and Retrieval-Augmented Generation (RAG) systems through hands-on Jupyter notebooks.

**Two main learning tracks:**
- **Week 1**: Medical Assistant (RAG system over Merck Manuals PDF)
- **Project 1**: Apple organizational innovation (multi-agent business case study)

## Environment Setup
- **Python version**: ≥3.10
- **Package manager**: `uv` (see [pyproject.toml](pyproject.toml))
- **Virtual environment**: `.venv/` (uv-managed, activate with `source .venv/bin/activate`)
- **Kernel**: `ai_biz_ut_austin` (Jupyter)

**To activate and verify:**
```bash
source .venv/bin/activate
python -m pip list | grep -E "langchain|openai|chromadb"
```

## Core Technology Stack

### LLM & Framework
- **LangChain**: 0.3.27 + langchain-community (primary agent framework)
- **LLM Provider**: OpenAI API (custom endpoint via `config.json`)
- **Tracing**: LangSmith for debugging agent chains

### Data & Storage
- **Vector DB**: ChromaDB 1.5.9 (primary), FAISS 1.14.2 (fallback)
- **Embeddings**: sentence-transformers 5.5.1
- **Document Processing**: PyMuPDF (PDF extraction), LangChain TextSplitter (chunking)
- **Data**: pandas, Hugging Face Datasets

### Configuration
- **API Keys**: `config.json` (contains OpenAI endpoint, LangSmith API key, Tavily API key)
- **Note**: `config.json` is `.gitignore`d; never commit credentials

## Key Patterns & Conventions

### Notebook Structure
- **Markdown-heavy**: Each notebook includes extensive problem statements and business context before code cells
- **Sequential execution**: Cells depend on prior cell output; kernel restart may be needed after package installs
- **Two-track design**: "Full Code" notebooks contain complete solutions; "Low Code" versions scaffold student learning
- **Cell organization**: Problem statement → imports → setup → RAG pipeline → evaluation → results

### RAG Pipeline (Common Pattern)
All RAG notebooks follow this sequence:
1. **Load documents**: Use `PyMuPDFLoader` or `TextLoader` for files
2. **Chunk text**: Use `RecursiveCharacterTextSplitter` with overlap
3. **Create embeddings**: Use `SentenceTransformerEmbeddings` or `OpenAIEmbeddings`
4. **Store vectors**: Store in `ChromaDB` or `FAISS`
5. **Build retrieval chain**: Use `RetrievalQA` from langchain
6. **Query & evaluate**: Test with business metrics or custom evaluation

### Agent Pattern (Business Applications)
- Agents use `langchain.chains.RetrievalQA` for information retrieval
- Agents are evaluated using:
  - Business metrics (accuracy, relevance)
  - LLM-based evaluation with `langchain_openai.ChatOpenAI`
  - Human-in-the-loop feedback

### Debugging Agent Chains
- Check LangSmith traces: Enable `LANGSMITH_TRACING` in `config.json`
- Inspect intermediate outputs: Print `result["source_documents"]` to see retrieved context
- Verify embeddings: Compare `embed(query)` with stored vectors using cosine similarity

## Debugging & Troubleshooting

**Import errors after environment changes?**
```bash
deactivate && source .venv/bin/activate
python -m pip list  # Verify packages
```

**Notebook kernel not showing?**
```bash
python -m ipykernel install --user --name ai_biz_ut_austin --display-name "Python (ai_biz_ut_austin)"
```

**LLM API failures?**
- Verify `config.json` contains valid `OPENAI_API_BASE` and `OPENAI_API_KEY`
- Check LangSmith traces for request/response details
- Test with a simple `OpenAI().invoke("Say hello")` in a notebook cell

**ChromaDB persistence issues?**
- ChromaDB stores embeddings in-memory by default in notebooks
- For persistent storage, initialize with `Chroma(persist_directory="./chroma_db", ...)`
